## Lighting Issues in the Preview World

The world I was assigned, "コクーン水族美術館 Redive", has fairly dark lighting overall.\
~~The Vket team also made two rounds of fixes to the exhibit lighting and display issues, but the overall lighting was still a little dark.~~

Even at a perfectly normal brightness in Unity,\
once I entered the Preview World...

it was way, WAY darker than expected.

I later learned from a senior student that\
lilToon's **Base Settings** include several options that control how the model receives light.

<StepGallery arrows={false} maxWidth="600px" steps={[
  { img: '/vket-diary/img/web-033.webp', label: 'lilToon Default Base Settings' }
]} />

| Setting | Purpose |
| ---------------------------------- | --------------------------------------------- |
| Lower Brightness Limit | Controls the model's minimum brightness after lighting is applied. Raising it prevents the model from becoming too dark in dim areas. |
| Upper Brightness Limit | Controls the model's maximum brightness after lighting is applied. Raising it preserves stronger lighting; lowering it helps prevent overexposure. |
| Monochrome Lighting | Controls how much the light's color gets desaturated. Higher values reduce how strongly red/blue environmental light tints the model. |
| Environment Strength on Shadow | Controls how much ambient light affects shadow colors. Higher values let shadows brighten with the surrounding light. |

The brightness differences caused by these settings may not be fully reproduced in the Unity submission scene,\
but once you're in the Preview World, they become extremely obvious...

Here's a comparison of three configurations:

In the end, all I could do was repeatedly tweak, re-upload, and test\
until the brightness felt right for the venue's environment.

> ※ The values above are only what worked for this particular venue — they won't fit every scene.
> ※ The submission scene's lighting differs from the actual venue, so always judge the final result in the Preview World.\
> ※ Pushing brightness values like Upper Brightness Limit too high can cause overexposure or display issues in brightly lit scenes — adjust to your venue.\
> ※ After each re-submission, it takes roughly 5 hours to half a day for the updated model to appear in the Preview World. Actual times may vary by venue, update batch, or system conditions — this is just my personal experience.

After pressing Create Backup,\
the system creates a ZIP file stored in VCC's `ProjectBackups` folder.

I happen to back up regularly anyway,\
so I dodged this problem by pure luck.

But here's what I found: after you hit "Submit" and VketTools finishes its chain of checks and processing,\
the moment the "Submission completed" window pops up, the settings under your booth number are instantly cleared...

<StepGallery arrows={false} maxWidth="600px" steps={[
  { img: '/vket-diary/img/web-035.webp', label: 'Submission completed' }
]} />

<StepGallery steps={[
{ img: '/vket-diary/img/web-042.webp', label: 'Inspector Settings Before Submission' },
{ img: '/vket-diary/img/web-037.webp', label: 'Inspector Settings After Submission' },
]} />

If you didn't create a backup before submitting,\
then the moment anything needs fixing and you have to re-submit, you'll be redoing every setting from scratch... Orz

To restore the pre-submission state, use the backup you created earlier.

Close Unity first, unzip the backup ZIP,\
then copy its contents into the original project folder and overwrite the existing files. This restores your pre-submission settings.

So please, seriously: **Backup first. Then submit.**

> ※ I haven't confirmed with other exhibitors whether this happens to everyone — this is just what happened to me.

## Texture Settings & Compression
### Texture Compression: Use Crunch Compression

Remember to check whether **Use Crunch Compression** is enabled on your textures.

Vket 2026 Summer has AssetBundle Build Size limits:\
3.00 MB for Item exhibits, 10.00 MB for Space exhibits.

You can check the AssetBundle file size after it is built with VketTools' Build Size Check.

Among the various things that affect file size, textures are usually one of the most noticeable contributors.\
Because of this, enabling or disabling Crunch Compression can make a very noticeable difference in the final file size.

<StepGallery arrows={false} steps={[
{ img: '/vket-diary/img/web-047.webp', label: 'Without Crunch Compression' },
{ img: '/vket-diary/img/web-048.webp', label: 'With Crunch Compression' },
]} />

That said, Crunch Compression is lossy — it can introduce block artifacts, blurriness, or detail loss.\
So you'll need to weigh file size against visual quality based on your textures and how they actually look.

## Other Small Issues

### FBX Mesh Settings

Some VRChat SDK checks, scripts, or model features\
need to read mesh data at runtime.

So after importing an FBX, select the file,\
and in the Inspector's Model tab,\
check the settings for Read/Write Enabled and Legacy Blend Shape Normals.

<StepGallery arrows={false} maxWidth="500px" steps={[
  { img: '/vket-diary/img/web-050.webp', label: 'FBX Mesh Settings' }
]} />

For meshes used by Avatars, make sure Read/Write Enabled is ON —\
otherwise you may get SDK check errors, features that do not work correctly, or failed uploads.

Meshes used in Worlds don't always need it —\
only enable it when a script or feature actually needs to read the mesh data.

**Legacy Blend Shape Normals** affects how normals are computed for Blend Shapes.\
There's no universally correct setting — judge by how the model actually renders.\
If deformations produce shading or normal artifacts, try toggling it and compare.


### Materials & SetPass Calls
Transparent materials (2-Pass Transparent), fur materials (Fur), outlines (Outline), and similar effects\
require additional rendering passes, so they also tend to increase SetPass Calls.

For example, an object with a single material starts with 1 SetPass Call;\
turn on Outline and — even though the material count hasn't changed — SetPass Calls become 2.
<StepGallery arrows={false} steps={[
{ img: '/vket-diary/img/web-051.webp', label: 'Without Outline' },
{ img: '/vket-diary/img/web-052.webp', label: 'With Outline' },
]} />

So don't judge by material count alone —\
always verify the actual SetPass Calls with VketTools' SetPass Check.
