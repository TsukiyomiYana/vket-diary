## テクスチャ設定と圧縮
### テクスチャ圧縮 : Use Crunch Compression

テクスチャで **Use Crunch Compression** が有効になっているか確認しましょう。

Vket 2026 Summer では AssetBundle の Build Size に上限があります：\
Item 出展は 3.00 MB、Space 出展は 10.00 MB。

AssetBundle をビルドしたあとのファイルサイズは、VketTools の Build Size Check で確認できます。

ファイルサイズに影響するさまざまな要素の中でも、テクスチャは特に影響が大きい部分です。\
そのため、Crunch Compression の有無による最終的なファイルサイズの差も非常にはっきり出ます。

<StepGallery arrows={false} steps={[
{ img: '/vket-diary/img/web-047.webp', label: 'Without Crunch Compression' },
{ img: '/vket-diary/img/web-048.webp', label: 'With Crunch Compression' },
]} />

ただし、Crunch Compression は非可逆圧縮なので、ブロックノイズやぼやけ、\
ディテールの損失が起きる可能性があります。\
テクスチャの内容と実際の表示を見ながら、ファイルサイズと画質のバランスを取る必要があります。

### 元画像サイズと Max Size

元画像の解像度以外、下記2枚のテクスチャの Unity インポート設定は完全に同じです。
* 4096 px の元画像を使用し、Unity の Max Size を 2048 px に設定
* 2048 px の元画像を使用し、Unity の Max Size を同じく 2048 px に設定

実際に比較すると、元から 2048 px の画像を使ったバージョンのほうが、\
4096 px の元画像を Unity で 2048 px に縮小したバージョンより明らかにシャープです。

## その他の小さな問題

### FBX モデルの Mesh 設定

一部の VRChat SDK チェック、スクリプト、モデルの機能は、\
実行時に Mesh データを読み取る必要があります。

そのため、FBX をインポートしたらファイルを選択し、\
Inspector の Model タブで、\
Read/Write Enabled と Legacy Blend Shape Normals の設定を確認しましょう。

<StepGallery arrows={false} maxWidth="500px" steps={[
  { img: '/vket-diary/img/web-050.webp', label: 'FBX Mesh Settings' }
]} />

Avatar 用の Mesh の場合、Read/Write Enabled が ON になっていることを確認してください。\
OFF だと SDK チェックのエラー、機能の不具合、アップロード失敗の原因になることがあります。

World 用の Mesh は必ずしも ON にする必要はなく、\
スクリプトや機能が Mesh データを読み取る場合のみ有効にすれば OK です。

**Legacy Blend Shape Normals** は Blend Shape の法線計算方法に影響します。\
決まった正解はなく、モデルの実際の表示結果で判断します。\
変形後に影や法線の異常が出る場合は、設定を切り替えて効果を比較してみましょう。


### マテリアルと SetPass Calls
透明マテリアル（2-Pass Transparent）、ファーマテリアル（Fur）、アウトライン（Outline）などのエフェクトは、\
追加のレンダリング処理が必要になるため、通常 SetPass Calls も増加します。

例えば、マテリアル1つだけのオブジェクトの SetPass Calls は元々 1 ですが、\
Outline を有効にすると、マテリアル数が変わらなくても SetPass Calls は 2 になります。
<StepGallery arrows={false} steps={[
{ img: '/vket-diary/img/web-051.webp', label: 'Without Outline' },
{ img: '/vket-diary/img/web-052.webp', label: 'With Outline' },
]} />

なので、マテリアルの数だけで判断せず、\
必ず VketTools の SetPass Check で実際の SetPass Calls を確認しましょう。
