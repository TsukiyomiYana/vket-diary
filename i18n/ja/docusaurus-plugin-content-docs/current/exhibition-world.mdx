出力前に、以下を確認しておきましょう :
* モデルのサイズ、向き、法線、オブジェクト名が正しいか。

### Unity アセットの整理

Unity にインポートしたら、個人的な習慣として、ファイルの用途ごとにフォルダを整理しています :
* Materials : マテリアル
* Models : FBX モデル
* Prefabs : Prefab
* Scenes : Unity シーンファイル
* Scripts : スクリプト
* Textures : テクスチャ (image)

次に、モデルやテクスチャなどのアセットを対応するフォルダに入れて、\
Materials に lilToon マテリアルを作成し、シーン内の対応するモデルに適用します。

これで基本的なモデルのインポートとマテリアル設定は完了です。

## インタラクティブ機能

今回のインタラクティブ機能は、ほとんどすべて [LuraSwitch2](https://booth.pm/ja/items/1969082) で作りました。

主に使った機能：
* ミラー切り替え
* オブジェクトの位置切り替え（Position A / Position B）
* Pickup オブジェクトのリスポーン

### LuraSwitch2 のインストール方法
[LuraSwitch2](https://booth.pm/ja/items/1969082) のインストールは簡単です。\
BOOTH から ZIP をダウンロードし、中の Unity Package をそのまま Unity プロジェクトにインポートします。

Assets に `QuickBrown` フォルダが現れたら、インポート成功です。

詳しい使い方は、\
公式の [Lura Switch 2 説明書](https://lurathunder.github.io/lura-switch-docs/) を直接読むことを強くおすすめします。

以下では、実際に使い、独自に調整した部分だけを記録します。

### ミラーの3段階切り替え（OFF / LQ / HQ）

[LuraSwitch2](https://booth.pm/ja/items/1969082) には最初からミラーの3段階切り替え機能があります。\
ただ、デフォルトの操作パネルは、複数のスイッチとレバーで構成されています。

プレイヤーが自由に調整できる反面、\
私は操作をシンプルにしたい派なので、ボタン1つでミラーの状態を順番に切り替えられるようにしたいと思いました。

<StepGallery steps={[
  { img: '/vket-diary/img/web-066.webp', label: 'LuraSwitch2 Default' },
  { img: '/vket-diary/img/web-067.webp', label: 'Single-Button Cycle' },
]} />

メインの `Switch_Mirror` スイッチは残しつつ、**Switch Mode** を **External** モードに変更しました。

[LuraSwitch2](https://booth.pm/ja/items/1969082) の元の Script と構造には手を加えず、\
既存のボタンとイベント呼び出しの仕組みをそのまま使う前提で、\
ミラーの3つの状態を管理する `MirrorCycleSwitch.cs` を別に書きました。

`MirrorCycleSwitch.cs` が主に制御するもの：
* MirrorActivator
* LQ_Activator
* HQ_Activator

スイッチを押すたびに、\
Script が次の状態に切り替わり、ミラーの表示品質とスイッチパネルの文字も同時に更新されます。

これで、本来なら複数のスイッチとレバーで操作する OFF / LQ (透明) / HQ (高画質) の切り替えが、\
ボタン1つで順番にサイクルできるようになりました。

ミラーの切り替えは **Local** に設定してあるので、\
プレイヤーごとに自分だけのミラー表示を切り替えられて、他の人には影響しません。

### Pickup リスポーン

展示スペースには、持ち上げて投げられるスティウミが1匹います。\
シーンの外に投げられて行方不明にならないように、\
元の位置に戻れる仕組みをシーン内に用意する必要がありました。

[LuraSwitch2](https://booth.pm/ja/items/1969082) には、オブジェクトを初期位置に戻す Respawn 機能が最初からあります。\
シンプルな手動リセットだけでいいなら、元の機能で十分です。

ただ、手動リセットに加えて、\
スティウミが遠くに投げられたときの自動リスポーンも欲しかったのです。

そこで `PickupRespawner.cs` を別に書いて、手動リセットと自動リスポーン機能を追加しました。\
[LuraSwitch2](https://booth.pm/ja/items/1969082) のボタンインターフェースをそのまま使い、**External** 経由でこの Script を呼び出します。

**Switch Mode** を **External** に設定し、スイッチを押したときに `RespawnAll` を呼ぶようにすれば、\
指定した Pickup オブジェクトが初期位置に戻ります。

自動リスポーンは、3秒ごとに Pickup オブジェクトの位置をチェックします。\
オブジェクトが初期位置から8メートルを超えて離れていて、かつプレイヤーに持たれていない場合、自動的に元の位置に戻ります。

複数のプレイヤーが同時に同じ Pickup をリスポーンさせないように、\
自動リスポーンはそのオブジェクトの現在のオーナーだけが処理します。

> ※ 8メートルのリスポーン距離と3秒のチェック間隔は、どちらも Inspector で自由に調整できます。\
※ 空気壁を厚くすれば、外に投げ出されるのを完全に防げるのは知っています...~~でも、正直みんなに押し出しチャレンジしてほしいんですよね www~~

## 初めての Skybox

Unity の設定がだいたい終わって、ふと振り返ると...

~~えっと...この Skybox、なんとかしないとダメみたいですね。~~

当然、~~Skybox を描いた経験なんてありません。~~\
ネットでもこのスペースに合う Skybox は見つからず、考えた結果...

~~自分で産むしかないみたいです~~。

少し調べたところ、Unity で Skybox を作る方法はいろいろあり、\
組み込みの Skybox Shader は大きく4種類に分けられます：
* [6 Sided](https://docs.unity3d.com/2023.1/Documentation/Manual/shader-skybox-6sided.html) : 前後左右上下に対応する6枚の正方形画像を使用。
* [Cubemap](https://docs.unity3d.com/6000.0/Documentation/Manual/shader-skybox-cubemap.html) : 6方向の画面を含む Cubemap アセットを使用。
* [Procedural](https://docs.unity3d.com/2022.3/Documentation/Manual/shader-skybox-procedural.html) : パラメータで空の色、太陽の位置、大気効果を調整。
* [Panoramic](https://docs.unity3d.com/2021.3/Documentation/Manual/shader-skybox-panoramic.html) : 1枚の360°パノラマ画像を空の背景として使用。

というわけで、自分にとって一番直感的な方法として、\
Blender の UV Sphere を使って **Panoramic Skybox** テクスチャを作ることにしました。

### Blender で Skybox テクスチャを作る方法

まず、Blender で **UV Sphere** を1つ作成します。\
`Segments` と `Rings` をどちらも `64` に、`Radius` を `8 m` に設定します。

次に Edit Mode に入り、`A` で全ての面を選択、\
`Alt + N` を押して **Flip** で法線を反転します。

この UV Sphere は、主に Skybox の見え方をシミュレートするためのものです。\
Skybox は球体の内側から外に向かって見るものなので、\
球の内側を正面にしておくと、内部から直接テクスチャの見え方を確認できます。

  { img: '/vket-diary/img/web-069.webp', label: 'Set Segments, Rings & Radius' },
  { img: '/vket-diary/img/web-070.webp', label: 'Flip Normals' },

続いて **UV Editing** ワークスペースに切り替え、`4096 × 2048`、比率 `2:1` の画像を作成します。\
ここでは `SkyboxMAP` と名付けました (名前はお好みで)。

UV Sphere にマテリアルを作成し、`SkyboxMAP` をマテリアルに割り当てます。

UV Sphere は作成時点で、パノラマ画像に対応する UV を持っています。\
そのため、テクスチャが球形空間の中でどんな位置・比率・歪みになるか、そのまま確認できます。

そのあと **Texture Paint** ワークスペースに切り替え、まず大まかな構図とガイド線を描き、\
画像を Photoshop や CSP に書き出してディテールを仕上げます。

### Unity の Skybox 設定

Skybox テクスチャが完成したら、画像を Unity にインポートし、新しい Material を作成します。\
ここではマテリアルを `Sky_Pastel` と名付けました (名前はお好みで)。

`Sky_Pastel` を選択し、Inspector で Shader を\
: **Skybox > Panoramic** に設定します。\

次に、完成した `SkyboxMAP` を Spherical (HDR) のテクスチャスロットに入れます。
他の設定はデフォルトのままで OK です。

ここで、各項目の用途を簡単にまとめておきます：

| 項目 | 説明 |
| --- | --- |
| Tint Color | Skybox テクスチャ全体に色を重ねます。<br />白に設定すると、元のテクスチャの色が維持されます。 |

### Bake Lighting は必要?

シーン内に動かないオブジェクトが大量にあって、固定の影と間接光の効果を残したい、\
同時に Realtime Lighting の計算負荷を減らして、パフォーマンスをさらに最適化したい。\
そんなときは Bake Lighting を検討する価値があります。

ベイクされたライティング情報は Baked Lightmaps に記録され、\
動くオブジェクトは Light Probes と組み合わせて、周囲の環境光情報を取得できます。

今回の展示スペースは規模が小さく、複雑なライティングの必要もなかったので、\
明暗や影の効果はほぼすべてテクスチャに直接描き込み、ライトベイクは行いませんでした。

必要な方は、自分のシーンの規模とパフォーマンス要件に合わせて、使うかどうか判断してください。
