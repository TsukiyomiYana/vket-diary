## 互動功能

這次絕大部分的互動功能、都是使用 [LuraSwitch2](https://booth.pm/ja/items/1969082) 製作的。

主要使用到的功能：
* 鏡子切換
* 物件位置切換（Position A / Position B）
* Pickup 物件回歸

### LuraSwitch2安裝方式
[LuraSwitch2](https://booth.pm/ja/items/1969082) 的安裝方式很簡單。\
從 BOOTH 下載 ZIP 檔後、將裡面的 Unity Package 直接匯入 Unity 專案。

Assets 中出現 `QuickBrown` 資料夾、就表示已經成功匯入了。

詳細的使用方式、\
強烈建議直接閱讀官方的 [Lura Switch 2 説明書](https://lurathunder.github.io/lura-switch-docs/)。

以下只記錄我實際使用、並另外做過調整的部分。

### 鏡子的三段式切換（OFF / LQ / HQ）

[LuraSwitch2](https://booth.pm/ja/items/1969082) 本身就提供了鏡子的三段式切換功能、\
不過預設的操作介面、是由多個開關與操作桿組成的。

這樣雖然提供了讓玩家自行調整的自由度、\
但我比較喜歡簡化操作、希望玩家只需要按一個按鈕、就能依序切換鏡子的狀態。

<StepGallery steps={[
  { img: '/vket-diary/img/web-066.webp', label: 'LuraSwitch2 Default' },
  { img: '/vket-diary/img/web-067.webp', label: '改為單按鈕循環切換' },
]} />

我保留了主要的 `Switch_Mirror` 開關、並將 **Switch Mode** 改為 **External** 模式。

在不修改 [LuraSwitch2](https://booth.pm/ja/items/1969082) 原有 Script 與結構、\
並沿用原本按鈕與事件呼叫機制的前提下、\
另外寫了一個 `MirrorCycleSwitch.cs`、負責管理鏡子的三種狀態。

`MirrorCycleSwitch.cs` 主要控制：
* MirrorActivator
* LQ_Activator
* HQ_Activator

每次按下開關時、\
Script 就會切換至下一個狀態、同時更新鏡子的顯示品質、以及開關面板上的文字。

這樣就能將原本需要透過多個開關與操作桿控制的 OFF / LQ (透明) / HQ (高清)、\
改成只使用單一按鈕、就能依序循環切換。

鏡子的切換設定為 **Local**、\
因此每位玩家都可以自行切換鏡子的顯示狀態、不會影響其他玩家。

### Pickup 回收

展示空間裡有一隻臭豆腐海兔、可以拿起來、丟出去。\
為了避免它被丟到場景外面後找不回來、\
我需要在場景內加入一個能讓它回到原位的機制。

[LuraSwitch2](https://booth.pm/ja/items/1969082) 本身就有讓物件回到初始位置的 Respawn 功能、\
如果只需要單純的手動重置、直接使用它原本的功能就足夠了。

不過、除了手動重置以外、\
我還希望臭豆腐海兔在被丟得太遠時可以自動回收。

因此、我另外寫了 `PickupRespawner.cs`、加入手動重置與自動回收功能。\
並沿用 [LuraSwitch2](https://booth.pm/ja/items/1969082) 的按鈕介面、透過 **External** 呼叫這支 Script。

將 **Switch Mode** 設為 **External**、並設定在按下開關時呼叫 `RespawnAll`、\
就能讓指定的 Pickup 物件回到初始位置。

自動回收則會每隔 3 秒檢查一次 Pickup 物件的位置。\
當物件距離初始位置超過 8 公尺、且沒有被玩家拿在手上時、就會自動回到原位。

為了避免多人同時對同一個 Pickup 物件執行回收、\
自動回收只會由該物件目前的擁有者負責處理。

> ※ 8 公尺的回收距離與 3 秒的檢查間隔、都可以在 Inspector 中自行調整。\
※ 我知道把空氣牆做厚一點、就可以完全避免它被丟出去...~~但其實我滿想讓大家試著把它擠出去 www~~

稍微查了一下資料、發現 Unity 製作 Skybox 的方式有很多種。\
內建的 Skybox Shader 大致可以分成以下四種：
* [6 Sided](https://docs.unity3d.com/2023.1/Documentation/Manual/shader-skybox-6sided.html) : 使用六張正方形圖片、分別對應前、後、左、右、上、下。
* [Cubemap](https://docs.unity3d.com/6000.0/Documentation/Manual/shader-skybox-cubemap.html) : 使用一個包含六個方向畫面的 Cubemap 資產。
* [Procedural](https://docs.unity3d.com/2022.3/Documentation/Manual/shader-skybox-procedural.html) : 透過參數調整天空顏色、太陽位置與大氣效果。
* [Panoramic](https://docs.unity3d.com/2021.3/Documentation/Manual/shader-skybox-panoramic.html) : 使用一張 360° 全景圖片作為天空背景。

於是、我決定選擇對自己來說最直覺的方式、\
利用 Blender 的 UV Sphere 來製作 **Panoramic Skybox** 貼圖。

### 如何用 Blender 製作 Skybox 貼圖

首先、在 Blender 中建立一顆 **UV Sphere**、\
將 `Segments` 與 `Rings` 都設為 `64`、並將 `Radius` 設為 `8 m`。

接著進入 Edit Mode、按下 `A` 全選所有面、\
再按下 `Alt + N` 、選擇 **Flip** 反轉法線。

這顆 UV Sphere 主要是用來模擬 Skybox 的觀看效果。\
由於 Skybox 是從球體內側向外觀看、\
因此需要將球體內側設為正面、方便直接從內部確認貼圖效果。

  { img: '/vket-diary/img/web-069.webp', label: '設定 Segments、Rings 與 Radius' },
  { img: '/vket-diary/img/web-070.webp', label: 'Normal Flip 反轉法線' },

接著切換到 **UV Editing** 工作區、建立一張 `4096 × 2048`、比例為 `2:1` 的圖片、\
這裡先將圖片命名為 `SkyboxMAP` (可以依照自己的習慣命名)。

在 UV Sphere 上建立材質、並將 `SkyboxMAP` 指定到材質中。

UV Sphere 建立時、已經帶有可以對應全景圖片的 UV。\
因此、可以直接用來確認貼圖在球形空間中的位置、比例與變形。

之後切換到 **Texture Paint** 工作區、先大致畫出想要的構圖與定位標線、\
再將圖片匯出到 Photoshop 或 CSP 中完成細節。

### Unity 的 Skybox 設定

完成 Skybox 貼圖後、將圖片匯入 Unity、並建立一個新的 Material。\
這裡將材質命名為 `Sky_Pastel` (可以依照自己的習慣命名)。

選取 `Sky_Pastel`、在 Inspector 中將 Shader 設為\
: **Skybox > Panoramic**\

接著、將完成的 `SkyboxMAP` 放到 Spherical (HDR) 的貼圖欄位中。
其他設定維持預設即可。

### 我需要 Bake Lighting 嗎?

當場景中有大量不會移動的物件、又希望保留固定的陰影與間接光效果、\
同時減少 Realtime Lighting 的運算負擔、進一步最佳化平台效能時、\
就可以考慮使用 Bake Lighting。

烘焙後的光照資訊會被記錄在 Baked Lightmaps 中、\
會移動的物件則可以搭配 Light Probes、取得周圍的環境光資訊。

這次的展示空間規模較小、也沒有複雜的光影需求、\
絕大部分的明暗與陰影效果、我都直接繪製在貼圖上、因此就沒有另外進行光照烘焙。

有需要的人、可以再依照自己的場景規模與效能需求、自行斟酌是否使用。
