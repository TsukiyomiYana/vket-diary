後來詢問學長後才知道、\
lilToon 的 **Base Settings**、有幾個可以調整模型受光效果的設定。

<StepGallery arrows={false} maxWidth="600px" steps={[
  { img: '/vket-diary/img/web-033.webp', label: 'lilToon Default Base Settings' }
]} />

| 設定 | 用途 |
| ---------------------------------- | --------------------------------------------- |
| Lower Brightness Limit | 控制模型受到光照後的最低亮度。提高數值、可以避免模型在暗處變得太黑。 |
| Upper Brightness Limit | 控制模型受到光照後的最高亮度。數值提高、可以保留更強的光照；反之、則可以避免過曝。 |
| Monochrome Lighting | 控制燈光顏色的單色化程度。提高數值、可以減少紅光、藍光等強烈環境光對模型造成的染色。 |
| Environment Strength on Shadow | 控制環境光對影色的影響程度。數值越高、影色越容易隨著周圍的環境光變亮。 |

這些設定造成的亮度差異、在 Unity 的入稿用 Scene 裡不一定能完整重現、\
但實際進入下見ワールド之後、就會變得非常明顯...

下面分別是三種設定的比較：

{ img: '/vket-diary/img/web-030.webp', label: 'Emission Added: 0.2' },

最後只能反覆調整並重新上傳測試、慢慢把亮度調整到適合展區環境的程度。

> ※ 上述數值調整只針對這次展區環境進行測試的結果、並非所有場景都適用。
> ※ 入稿用 Scene 的光照環境和實際會場不同、最終效果還是要以下見ワールド為準。\
> ※ Upper Brightness Limit 等亮度相關的數值如果拉太高、可能會在光照較強的場景中造成過曝或顯示異常、需要根據展區環境調整。\
> ※ 每次重新入稿後、約需等待 5 小時至半天、更新後的模型才會反映到下見ワールド。實際時間可能因展區、更新批次或系統狀況而不同、這裡僅記錄我自己的體感。

## 貼圖（Texture）設定與壓縮
### 貼圖壓縮 : Use Crunch Compression

記得確認貼圖是否有開啟 **Use Crunch Compression**。

Vket 2026 Summer 對 AssetBundle 的 Build Size 設有上限：\
Item 出展為 3.00 MB、Space 出展則為 10.00 MB。

可以透過 VketTools 的 Build Size Check 來確認 AssetBundle 打包後的檔案大小。

而在影響檔案大小的各種項目中、貼圖通常是最明顯的一部分。\
因此、貼圖是否開啟 Crunch Compression、最後造成的檔案大小差距也會非常明顯。

<StepGallery arrows={false} steps={[
{ img: '/vket-diary/img/web-047.webp', label: 'Without Crunch Compression' },
{ img: '/vket-diary/img/web-048.webp', label: 'With Crunch Compression' },
]} />

不過、Crunch Compression 畢竟是屬於有損壓縮、可能會造成色塊、模糊或細節流失。\
因此、仍需要根據貼圖內容與實際顯示效果、在檔案大小和畫面品質之間做出取捨。

### 原始圖片尺寸與 Max Size

除了原始圖片尺寸以外、兩張貼圖在 Unity 裡的其他匯入設定完全相同。
* 使用 4096 px 原圖、並將 Unity 的 Max Size 設為 2048 px
* 使用 2048 px 原圖、並將 Unity 的 Max Size 同樣設為 2048 px

實際比較可以發現、直接使用 2048 px 原圖的版本、\
明顯比使用 4096 px 原圖、再由 Unity 縮小至 2048 px 的版本更加清晰。

## 其他小問題

### FBX 模型檔的 Mesh 設定

部分 VRChat SDK 檢查、Script 或模型功能、\
需要在執行時讀取 Mesh 資料。

因此、匯入 FBX 模型檔後、記得選取該檔案、\
並在 Inspector 的 Model 分頁中、\
確認 Read/Write Enabled 與 Legacy Blend Shape Normals 的設定。

<StepGallery arrows={false} maxWidth="500px" steps={[
  { img: '/vket-diary/img/web-050.webp', label: 'FBX Mesh Settings' }
]} />

如果是 Avatar 使用的 Mesh、請確認 Read/Write Enabled 已經開啟、\
否則可能會導致 SDK 檢查報錯、相關功能無法正常運作、或無法上傳。

World 使用的 Mesh 則不一定需要開啟、\
只有相關 Script 或功能需要讀取 Mesh 資料時、才需要另外啟用。

**Legacy Blend Shape Normals** 則是會影響 Blend Shape 法線的計算方式。\
它沒有固定答案、需要依照模型實際的顯示結果來判斷。\
如果變形後出現陰影或法線異常、可以切換設定來比較一下效果。


### 材質球與 SetPass Calls
透明材質（2-Pass Transparent）、毛髮材質（Fur）、外框（Outline）等效果、\
因為需要額外的渲染處理、通常也會增加 SetPass Calls。

例如、一個物件只套用了一個材質球時、SetPass Calls 原本是 1；\
但將 Outline 開啟之後、即使材質球數量沒有改變、SetPass Calls 也會增加為 2。
<StepGallery arrows={false} steps={[
{ img: '/vket-diary/img/web-051.webp', label: 'Without Outline' },
{ img: '/vket-diary/img/web-052.webp', label: 'With Outline' },
]} />

因此、不能只根據材質球數量來進行判斷、\
一定要透過 VketTools 的 SetPass Check 來確認實際產生的 SetPass Calls。
