後來詢問學長後才知道、\
lilToon 的 **Base Settings**、有幾個可以調整模型受光效果的設定。

<StepGallery arrows={false} maxWidth="600px" steps={[
  { img: '/vket-diary/img/web-033.webp', label: 'lilToon Base Settings' }

| 設定 | 用途 |
| ---------------------------------- | --------------------------------------------- |
| Lower Brightness Limit | 模型最低可以暗到什麼程度。數值提高、可以避免模型在暗處黑掉。 |
| Upper Brightness Limit | 模型最高可以亮到什麼程度。數值提高、可以保留更強的光照；反之、則可以避免過曝。 |
| Monochrome Lighting | 降低燈光本身的顏色。減少紅光、藍光等強烈環境光把模型染色的程度。 |
| Environment Strength on Shadow | 控制環境光影響陰影的程度。數值提高、陰影更容易受到周圍光線影響。 |

這些設定造成的亮度差異、在 Unity 的入稿用 Scene 裡不一定能完整重現、\
但實際進入下見ワールド之後、就會變得非常明顯...

## 貼圖（Texture）設定與壓縮
### 貼圖壓縮 : Use Crunch Compression

記得確認貼圖是否有開啟 **Use Crunch Compression**。

Vket 2026 Summer 對 AssetBundle 的 Build Size 設有上限：\
Item 出展為 3.00 MB、Space 出展則為 10.00 MB。

可以透過 VketTools 的 Build Size Check 來確認打包後的檔案容量。

貼圖有沒有開啟 Crunch Compression、最後的容量差距真的非常明顯。

<StepGallery arrows={false} steps={[
{ img: '/vket-diary/img/web-047.webp', label: 'Without Crunch Compression' },
{ img: '/vket-diary/img/web-048.webp', label: 'With Crunch Compression' },
]} />

雖然 Crunch Compression 屬於有損壓縮、可能會出現色塊或細節流失、
所以還是要在檔案容量與顯示品質之間做個取捨。

### 原始圖片尺寸與 Max Size

除了原始圖片尺寸以外、兩張貼圖在 Unity 裡的其他設定完全相同。
* 使用原圖 4096 px、在 Unity 的 Max Size 設為 2048 px
* 使用原圖 2048 px、在 Unity 的 Max Size 同樣設為 2048 px

但原圖本身就是 2048 px 的版本、\
明顯比使用 4096 px 原圖、再交給 Unity 縮小成 2048 px 的版本更加清晰。

## 其他小問題

### FBX 模型檔的 Mesh 設定

選擇 FBX 模型檔後、在 Inspector 的 Model 裡、\
記得確認 Read/Write Enabled 與 Legacy Blend Shape Normals 是否已經勾選。

<StepGallery arrows={false} maxWidth="500px" steps={[
  { img: '/vket-diary/img/web-050.webp', label: 'FBX Mesh Settings' }
]} />

部分 VRChat SDK、Script 或模型功能需要讀取 Mesh 資料、\
如果沒有開啟 Read/Write Enabled、可能會出現檢查錯誤、功能失效或無法順利上傳的狀況。

Legacy Blend Shape Normals 則會影響 Blend Shape 變形時的法線計算、\
如果模型有使用表情或其他 Blend Shape、也記得一起確認。

### 材質球與 SetPass Calls
透明材質（2-Pass Transparent）、毛髮材質（Fur）、外框（Outline）等效果、\
因為需要額外渲染一次、通常也會增加 SetPass Calls。

例如、一個物件只套用了一個材質球時、SetPass Calls 原本是 1；\
但將 Outline 開啟之後、即使材質球數量沒有改變、SetPass Calls 也會增加為 2。
<StepGallery arrows={false} steps={[
{ img: '/vket-diary/img/web-051.webp', label: 'Without Outline' },
{ img: '/vket-diary/img/web-052.webp', label: 'With Outline' },
]} />

所以不能只看使用了多少個材質球、\
一定要透過 VketTools 的 SetPass Check、來確認最後的 SetPass Calls 數值。
